Method Details

~ (void) on(events, callback, subscriber)

Set callback to be invoked when object notifies about events

Examples:

Render view when model changes: 

model.on 'change', view.render, view

Parameters:

  • events ( String ) — Event names separated by space
  • callback ( String|Function ) — Callback or its name which will be invoked
  • subscriber ( Object ) — Object which subscribes events. Subscriber will be used as context for callback

~ (void) once(events, callback, subscriber)

Version of on which works the same way, but callback will be invoked only once

Examples:

Initialize dependencies: 

app.once 'start', 'startRouting', router

Parameters:

  • events ( String )
  • callback ( String|Function )
  • subscriber ( Object )

~ (void) bind(events, callback, subscriber)

Version of on which works well with events specified as object

Examples:

Bind many events: 

model.bind
  positionChange: 'reorder'
  valueChange: 'updateValue'
, view

Parameters:

  • events ( String|Object )
  • callback ( String|Function )
  • subscriber ( Object )
